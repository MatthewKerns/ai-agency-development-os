# LinkedIn Content Creation Process

**Purpose:** Step-by-step workflow for creating high-value LinkedIn posts with infographics

**Time Investment:** ~2 hours per week for 3 posts
**Output:** 3 LinkedIn posts + 3 infographics + 3 process docs

---

## Weekly Workflow

### Monday Morning (30 min) - Plan Week's Topics

**Step 1: Review Calendar**
- Check `post-calendar.md` for scheduled topics
- Identify this week's 3 themes
- Ensure topics align with Developer Academy vision

**Step 2: Confirm Process Doc Topics**
- Each post needs a corresponding process doc
- Verify process docs are relevant to business owners
- Check that topics build on each other (if sequential)

**Step 3: Block Time**
- Tuesday AM: Write process docs (1.5 hrs)
- Wednesday AM: Create LinkedIn posts (1 hr)
- Thursday AM: Generate infographics (1 hr)
- Friday AM: Schedule posts (30 min)

---

## Tuesday: Write Process Docs (1.5 hours)

### For Each Process Doc (30 min each):

**Step 1: Choose Framework** (5 min)
- Requirements gathering framework
- Process mapping template
- Documentation checklist
- Developer handoff guide
- Automation blueprint structure
- Quality validation criteria

**Step 2: Write Deep Content** (20 min)
- Target: 500-1000 words
- Include specific examples
- Provide actionable frameworks
- Use bullet points and structure
- Add templates/checklists where relevant

**Step 3: Extract Key Points** (5 min)
- Identify 3-5 main takeaways
- Note visual elements for infographic
- Highlight quotable insights
- Mark framework/checklist sections

**Template Structure:**
```markdown
# [Process Doc Title]

## Overview
[What this process solves - 2-3 sentences]

## The Problem
[Why business owners struggle with this - 1 paragraph]

## The Framework
[Step-by-step approach - numbered list]

## Example
[Real-world application - 1-2 paragraphs]

## Common Mistakes
[What to avoid - bullet points]

## Quick Start
[Immediate action step - checklist]

## Key Takeaways
[3-5 bullets for infographic]
```

**Save Location:** `process-docs/[topic-name].md`

---

## Wednesday: Create LinkedIn Posts (1 hour)

### For Each Post (20 min each):

**Step 1: Read Process Doc** (3 min)
- Review full process document
- Identify the ONE main insight
- Extract most compelling example

**Step 2: Write Hook** (5 min)
- First line must stop the scroll
- Use proven formulas:
  - "Most business owners think [X], but..."
  - "Your developers are guessing. Here's why:"
  - "[Number] process documentation mistakes costing you $[X]"
  - "The [framework name] that saves [time/money]"

**Step 3: Write Body** (8 min)
- Paragraph 1: State the problem (pain point)
- Paragraph 2-3: Share the insight/framework
- Paragraph 4: Give actionable next step
- Keep total word count: 150-250 words

**Step 4: Write CTA** (2 min)
- Ask engagement question
- Invite comments/shares
- Connect to bigger vision
- Examples:
  - "What's your biggest process documentation challenge?"
  - "Have you seen this mistake in your projects?"
  - "Want the full framework? Drop a comment ðŸ‘‡"

**Step 5: Format** (2 min)
- Short paragraphs (2-3 lines max)
- Add line breaks for readability
- Use emojis sparingly (1-2 per post)
- Bold key phrases (use asterisks)
- Include relevant hashtags (3-5 max)

**Template:**
```markdown
---
**Post:** [Week X - Day]
**Topic:** [Main theme]
**Process Doc:** [Filename]
**Infographic:** [To be created]
**Scheduled:** [Date/Time]
---

[HOOK - Bold first line]

[Problem paragraph - establish pain]

[Insight paragraph 1 - share framework]

[Insight paragraph 2 - specific example or expansion]

[Action paragraph - what to do next]

[CTA - engagement prompt]

---
**Hashtags:** #ProcessDocumentation #DeveloperWorkflow #BusinessAutomation

**Infographic Points:**
- [Key point 1 from process doc]
- [Key point 2 from process doc]
- [Key point 3 from process doc]
```

**Save Location:** `posts/[YYYY-MM]/week-[X]/post-[X]-[day].md`

---

## Thursday: Generate Infographics (1 hour)

### For Each Infographic (20 min each):

**Step 1: Prepare NotebookLM Input** (5 min)
- Open corresponding process doc
- Copy key sections:
  - The Framework section
  - Key Takeaways
  - Quick Start checklist
- Paste into NotebookLM

**Step 2: Generate Visual** (10 min)
- Use NotebookLM to create infographic
- Request visual format:
  - Framework diagram
  - Step-by-step visual
  - Checklist format
  - Before/after comparison
- Ensure branding consistency
- Verify readability (mobile-friendly)

**Step 3: Export and Save** (5 min)
- Export as PNG (LinkedIn optimal)
- Name file: `week-[X]-post-[X].png`
- Save to: `infographics/`
- Update post file with infographic filename

**Infographic Guidelines:**
- Keep text minimal (visual focus)
- Use clear hierarchy
- Ensure mobile readability
- Include subtle branding
- Match LinkedIn's 1200x1200px format

---

## Friday: Schedule Posts (30 minutes)

**Step 1: Review All 3 Posts** (10 min)
- Read each post aloud
- Check for typos/errors
- Verify infographic references
- Confirm hashtags are relevant
- Test line breaks/formatting

**Step 2: Schedule in LinkedIn** (15 min)
- Post 1: Monday 9:00 AM
- Post 2: Wednesday 9:00 AM
- Post 3: Friday 9:00 AM
- Attach corresponding infographics
- Add alt text to images (accessibility)
- Double-check formatting

**Step 3: Track in Calendar** (5 min)
- Mark posts as "scheduled" in `post-calendar.md`
- Note any insights for next week
- Plan following week's topics

## Quality Checklist

### Before Scheduling ANY Post:

**Content Quality:**
- [ ] Hook stops the scroll (compelling first line)
- [ ] Speaks to business owners (not developers)
- [ ] Provides specific, actionable value
- [ ] Includes real example or framework
- [ ] Has clear next action step
- [ ] CTA prompts engagement
- [ ] Word count: 150-250 words
- [ ] No jargon or complex terms

**Format Quality:**
- [ ] Short paragraphs (2-3 lines)
- [ ] Line breaks for readability
- [ ] Bold/italic used sparingly
- [ ] Hashtags relevant (3-5 max)
- [ ] Infographic attached
- [ ] Alt text added
- [ ] Mobile preview checked

**Strategic Alignment:**
- [ ] Connects to process documentation theme
- [ ] Supports Developer Academy vision
- [ ] Builds on previous posts (if sequential)
- [ ] Provides standalone value
- [ ] Attracts target audience (business owners)

## Emergency Backup Process

**If You're Short on Time:**

**15-Minute Post Creation:**
1. Use existing process doc (3 min)
2. Extract one key framework (2 min)
3. Write 3-paragraph post (7 min)
4. Use simple text-based "infographic" (2 min)
5. Schedule immediately (1 min)

**Quality > Consistency:**
- Better to skip a post than publish low-quality content
- Reschedule if needed
- Focus on value delivery

## Performance Review Process

### After Each Post (5 min):
- Track engagement (likes, comments, shares)
- Note top-performing hooks
- Identify which topics resonate
- Save high-engagement posts as templates

### Weekly Review (15 min):
- Compare 3 posts' performance
- Identify patterns in engagement
- Adjust next week's topics based on data
- Update `performance-tracking.md`

### Monthly Review (30 min):
- Analyze month's overall performance
- Identify best post types
- Refine content strategy
- Plan next month's themes

## Tips for Maximum Engagement

### Hook Writing:
- Use numbers ("3 mistakes..." "5-step framework...")
- Ask questions ("Why do most business owners...")
- Make bold claims ("This changed everything about...")
- Create curiosity gaps ("Most people miss this...")

### Content Writing:
- Write like you talk (conversational)
- Use "you" and "your" (direct address)
- Break up long sentences
- Add specific examples
- Include frameworks/templates

### CTA Writing:
- Ask open-ended questions
- Invite personal experiences
- Request specific actions
- Create community feeling

## Troubleshooting

**Problem:** Running out of topic ideas
**Solution:** Review client pain points, network contacts insights, market trends from AUTOMATION-TYPES-MASTER-LIST.md

**Problem:** Posts not getting engagement
**Solution:** Review hooks, ensure speaking to business owners not developers, add more specific examples

**Problem:** Process docs taking too long
**Solution:** Use existing frameworks from this project, extract from real client work, start with templates

**Problem:** Infographics look unprofessional
**Solution:** Simplify design, focus on clarity over aesthetics, use NotebookLM templates
